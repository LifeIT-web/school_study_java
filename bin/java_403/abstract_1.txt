. Abstract Class(추상 클래스)
  
  : 추상 클래스는 설계도에 비유한다면 미완성 설계도라고 할수 있음
  	미완성 설계도란 완성되지 못한 설계도면을 말함
  	미완성 설계도로 완성 제품을 만들수 없듯이 추상 클래스도 인스턴스를 만들수 없음
  	추상 클래스는 상속을 통해서 자손 클래스에 의해서만 객체를 만들수 있음
  	
  	Abstract Method 와 Abstract Class
  	
  	: 추상 메소드는 직접 어떠한 명령을 수행하는 것이 아니고 메소드의
  	   머리 부분만 정의해 놓은 것을 의미함
  	   추상 클래스는 추상 메소드를 한개 이상 포함하고 있는 클래스를 말함
  	   
  	   
  	   		. 추상 메소드 : 메소드의 머리부분만 정의해 놓은 메소드
  	   		. 추상 클래스 : 추상 메소드를 한개이상 포함하는 클래스
  	   		
  	  추상 클래스를 상속받은 클래스는 추상 메소드를 재정의해서 더 이상
  	  추상 메소드를 갖지 않는 클래스로 바꾸고 객체화함
  	  
  	  		. 추상 클래스의 역할
  	  		
  	  	. 작업 명세표를 제공
  	  	. 추상 클래스에 있는 추상 메소드를 반드시 구현하도록 강제함
  	  	
  	  	 추상 클래스와 추상 메소드의 사용
  	  	 
  	  	 	형식
  	  	 	
  	  	 abstract class 클래스 이름
  	  	 	{
  	  	 			abstract void 추상메소드이름();(특징 : 몸통은 없다)
  	  	 			...;
  	  	 	}
  	  	 	
  	  	 	
  	  	 	
  	 . 종단 클래스
  	 
  	 : 클래스를 선언할때 final 키워드를 사용하는 클래스를
  	    종단 클래스라고 함
  	    종단 클래스는 더 이상 상속 할수 없는 클래스 임
  	    (완벽하기 때문에 건들지 않겠다. 상속받지 않겠다는 뜻 -> ex) String, System)
  	    형식
  	    
  	    final class 클래스 이름
  	    {
  	    	...;
  	    }
