Layout

: 지금까지 컴포넌트를 생성하고 컨테이너에 등록해
   출력하는 방법을 알아 봄. 그런데 문제가 있음
   컨테이너에 컴포넌트를 출력할때 어떻게 배치할 것인지를
   지정하지 않음. 자바는 일정한 모양으로 컴포넌트를
   재 배치 할 수 있도록 하기 위해 Layout Manager 를
   제공함. 레이아웃은 컴포넌트를 어떻게 배치할 것인가를
   정하는 규격이고 이를 설정하는 것이 레이아웃 메니저임
   
   자바에서 컴포넌트 위치를 결정하는 방법은 두가지 있음
   
   1. 컨테이너에 add() 메소드를 사용하는 순서에 따라 위치가
   		결정됨
   		
   2. 레이아웃 메니저에 따라 위치가 결정됨
   		ex) South, North... 등
   
       자바의 Container는 컴포넌트의 일종이기 때문에 컨테이너에 
       또다른 컨테이너를 등록 할수 있음
       컨테이너는 프레임, 패널, 윈도우, 애플릿 등이 있음
       결국 레이아웃 메니저는 컨테이너를 포함한 모든 컴포넌트의 
       배치에 관여함
       자바에서 제공하는 레이아웃 관리자는
   FlowLayout, GridLayout, BorderLayout, CardLayout 등이 있음
   
   	레이아웃 메니저는 컴포넌트를 어떻게 배치하는지를 결정하는 클래스임
   	그렇기 때문에 컨테이너를 제외한 모든 컴포넌트의 위치에 관여함
  	컨테이너에 또다른 컨테이너가 등록되어 있다면 서브 컨테이너도
  	레이아웃 메니저에 의해 위치가 결정됨
